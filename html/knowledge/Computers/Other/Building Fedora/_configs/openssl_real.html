OpenSSL Configuration using a real CA Cert

a. Download the CA Certificate Anyone who is going to sign your certificate, is going to make a CA Certificicate available to you. Find this cert and place it in the directory /usr/share/ssl/certs/ naming it CA.crt Openldap Cert<--> e. Generate a Certificate Signing Request First we generate an private key
cd /usr/share/ssl/private && /usr/bin/openssl genrsa -out hawaii.efficax.net.key 1024
Using this key, we generate a Certificate Request (CSR).
/usr/bin/openssl req -new -key hawaii.efficax.net.key -out hawaii.efficax.net.csr 
You are about to be asked to enter information that will be incorporated
into your certificate request.
What you are about to enter is what is called a Distinguished Name or a DN.
There are quite a few fields but you can leave some blank
For some fields there will be a default value,
If you enter '.', the field will be left blank.
-----

Country Name (2 letter required) [GB]: US
State or Province Name (full name) [Berkshire]: Texas
Locality Name (eg, city) [Newbury]: Cedar Park
Organization Name (eg, company) [My Company Ltd]: Efficax
Organizational Unit Name (eg, section) []:< cr >
Common Name (eg, your name or your server's hostname) []: hawaii.efficax.net

Email Address []: nagray@austin.rr.com
             
Please enter the following 'extra' attributes
to be sent with your certificate request
A challenge password []:< cr >
An optional company name []:< cr >
The key should now be moved to /usr/share/ssl/private and chmoded 0710. After you have done this, you will send the CSR to the Certificate Authority. They will return a certificate which you will name hawaii.efficax.net.crt and plaace in the /usr/share/ssl/certs directory.
