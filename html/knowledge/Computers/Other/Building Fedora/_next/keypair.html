Authentication via public/private keys

Purpose

Make it so that nick on the mac can log into the root account on the server and vice versa without a password

Allow Mac to login to Server

On Mac generate a key 
ssh-keygen -t dsa
Copy key over to root on Server 
scp /Users/nick/.ssh/id_dsa.pub root@192.168.122.10:.ssh/authorized_keys
Test 
ssh root@192.168.122.10
Copy key over to nick on Server 
scp /Users/nick/.ssh/id_dsa.pub nick@192.168.122.10:.ssh/authorized_keys
Test 
ssh nick@192.168.122.10

Allow Server to login to Mac

Login as root Generate a key 
ssh-keygen -t dsa
Copy key over to Mac 
scp /root/.ssh/id_dsa.pub nick@192.168.122.10:.ssh/authorized_keys
Test 
ssh nick@192.168.122.10
Exit root 
exit

Note this step

/usr/local/bin/note.sh keypair
