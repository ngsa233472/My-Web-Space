Spamassassin configuration

Purpose

Tags spam in the mail stream, adding a "X-Spam-Status" header and an X-Spam-Level: that can be used for sorting mail

Configure startup
/bin/systemctl enable spamass-milter
/bin/systemctl enable spamassassin

/bin/systemctl  start spamassassin
/bin/systemctl  start spamass-milter

/bin/systemctl  status spamassassin
/bin/systemctl  status spamass-milter

Note this step

/usr/local/bin/note.sh spamassassin_configuration
